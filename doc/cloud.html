Cloud Spawners¶

Cloud spawners run a distributed system by provisioning hosts in the cloud.

class dist_zero.spawners.cloud.aws.Ec2Spawner(system_id, aws_region='', base_ami='ami-1f057767', security_group='', instance_type='t2.micro')[source]¶

A Spawner subclass that creates each new MachineController instance by spinning up an aws ec2 instance and starting a long-running dist_zero.machine_init process on it.

Parameters:
  • system_id (str) – The id of the overall distributed system
  • aws_region (str) – The aws region in which to spawn the new ec2 instances.
  • base_ami (str) – A base aws ami image to use when spawning new ec2 instances.
  • security_group (str) – A security group to add to all new ec2 instances. It must open the appropriate ports. Ports for communication with running instances are defined in dist_zero.settings
  • instance_type (str) – The aws instance type (e.g. ‘t2.micro’)

send_to_machine(machine, message, sock_type='udp')[source]¶

Send a message from the current process to a MachineController listening on a port on a spawned machine.

Parameters:
  • machine (Handle) – The Handle of the MachineController for one of the managed machines.
  • message (Message) – Some json serializable message to send to that machine.
  • sock_type (str) – Either ‘udp’ or ‘tcp’. Indicating the type of connection.
Returns:

None if sock_type == ‘udp’. If sock_type == ‘tcp’, then return the response from the MachineController tcp API.
